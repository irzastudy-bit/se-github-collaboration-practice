# Code Review Checklist
Gunakan checklist berikut saat melakukan review Pull Request teman.
## Functionality
- [ ] Fitur berjalan sesuai kebutuhan.
- [ ] Tidak ada error saat program dijalankan.
- [ ] Output sesuai dengan skenario pengujian.
## Code Quality
- [ ] Nama variabel mudah dipahami.
- [ ] Struktur kode rapi.
- [ ] Tidak ada kode duplikat berlebihan.
- [ ] Fungsi memiliki tanggung jawab yang jelas.
## Collaboration
- [ ] Branch sudah sesuai format.
- [ ] Commit message jelas.
- [ ] Pull Request memiliki deskripsi lengkap.
## Contoh Komentar Review yang Baik
"Fitur sudah berjalan, tetapi validasi input status sebaiknya dibatasi
hanya todo, in_progress, dan done agar data lebih konsisten."
"Nama variabel sudah jelas. Namun, fungsi ini bisa dipisah agar lebih mudah
dibaca."

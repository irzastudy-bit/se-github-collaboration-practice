# Software Engineering - GitHub Collaboration Practice
Repo ini digunakan untuk praktikum kolaborasi kode menggunakan GitHub.
## Tujuan Praktikum
Mahasiswa mampu:
1. Membuat branch sesuai tugas fitur.
2. Melakukan commit dan push ke GitHub.
3. Membuat Pull Request.
4. Melakukan code review terhadap pekerjaan anggota tim.
5. Melakukan merge setelah review disetujui.
## Studi Kasus
Aplikasi sederhana To-Do List untuk mengelola task proyek software
engineering.
## Fitur yang Dikembangkan
- Menampilkan seluruh task
- Menambahkan task baru
- Mengubah status task
- Menghapus task
- Mencari task berdasarkan assignee
## Workflow Praktikum
1. Clone repository.
2. Buat branch baru.
3. Kerjakan fitur sesuai pembagian.
4. Commit perubahan.
5. Push branch ke GitHub.
6. Buat Pull Request.
7. Lakukan review pada PR teman.
8. Perbaiki kode jika ada komentar.
9. Merge setelah disetujui.
